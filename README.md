### 导入逻辑的改动
- 游戏内的占位图形改为**只允许使用基本图形中的矩形或圆形，不允许其他任何图形**，否则禁导入
- 游戏内的层数和软件内的Layer数量必须完全一样，否则禁导入
- 导入时，程序会告诉你检测到了多少个椭圆和多少个矩形，还会随机抽6个图层，将它们的数据放到屏幕上供你审查，你需要确认：
  
  1.椭圆102的数量等于你游戏内使用的圆形占位图层数量，矩形101的数量等于你在游戏内使用的矩形占位图层数量，并且这俩值加起来应该等于总的有效图层数量。
  
  2.RGBA的值符合你在游戏内看到的颜色和透明度。例如，如果你选的是个白色圆，RGBA应该全是255
  
  3.position、scale和angle是正常范围内的值（如果你没乱动图形，position和angle应该全是0.0，scale应该全是1.0）

  4.mask的值是0
  
- 确保这些数据都合规，才确认导入。如果任何数值不符合，千万不要导入！！随便输入点别的什么内容就可取消，或者直接把软件关了。
- 这么做仅仅是为了内存安全，我发现forza-painter的导入代码是采用“评分制”，并且只要求50%的内存数据符合特征，就认为可以导入。但是我在评论区看到有一个老哥反映“显示注入成功，但是游戏内什么都没有”，这恐怕是定位到了内存里的其他位置。毕竟FH6游戏的内存很大，产生错误定位的可能性不是完全没有，这样是有风险的
- 正因如此，vinylizer才会要求所有图层必须是椭圆或者矩形，并且还会随机抽样几个图层让你确认。这样就不可能出现错写内存的问题。
